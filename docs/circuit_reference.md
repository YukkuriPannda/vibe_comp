# 回路設計（Phase 1 確定版 / 2026-07-01）

**重要**: この回路はデータシートの式・KiCad標準シンボルのピン配置と照合して設計したオリジナル設計であり、既存の特定ペダル(Ross Compressor等)のコピーではない。**まだSPICEシミュレーション(Phase2)・ブレッドボード試作(Phase3)による実機検証を行っていない。** 基板発注(Phase6)の前に必ずPhase2/3を実施すること。特にコンプレッサーは帰還の極性を間違えると圧縮ではなく発振・エキスパンダー的動作になるため、試作確認は必須。

## 全体ブロック構成

```
入力Jack(J1,電池ON/OFF兼用)
  → 入力バッファ(U1A)
  → OTAゲイン制御段(IC1 LM13700 OTA#1、Sustainで帰還量を調整)
  → メイクアップゲイン段(U1B)
  → Levelポット(VR2)
  → 出力Jack(J2)

OTA出力ノード → エンベロープ検出(D1,C_hold等) → Sustainポット(VR1) → Q_ctrl(NPN) → IABC1(帰還でゲイン低下=圧縮)

Cherry MXスイッチ(SW1、手持ち部品) → デバウンス → TC4013 Dフリップフロップ(トグル)
  → Q_relay(NPN)駆動 → リレー(RL1, DPDT) → 信号経路をトゥルーバイパス切替
  → LED1(フリップフロップの状態を表示)
```

## 3. OTAゲイン制御段（IC1: LM13700 OTA#1）

**【2026-07-01 SPICE検証で追加修正】** LTspice + LM13700の実トランジスタレベルSPICEモデル(deanm1278/LM13700-spice-model, TIの公開回路を元にしたもの)で検証した結果、以下2点の設計不備が判明し修正した:

1. **入力にアッテネータが必要だった**: U1Aの出力(最大1.6Vpp程度を想定)をそのままOTAの差動入力に入れると、OTAの実用線形入力範囲(データシートFig.20の例で「1Vpの入力を33mVppまで減衰させる」と明記、リニアライズダイオード使用時でも60mVpp程度が上限)を100倍以上超えてしまい、圧縮が効く前にほぼ常時ハードクリップする計算になった。→ **U1A出力とOTA入力の間にアッテネータ(1/60程度)を追加**
2. **OTA差動入力のDCバイアス基準点が誤っていた**: 「pin3→GND直結」「pin4→GND直結」という当初設計は、SPICEモデル上で出力が全く出ない(ゲインほぼゼロ)結果になった。差動入力トランジスタがV−レール付近で電流不足になり事実上遮断されるため。→ **pin3・pin4とも基準をGNDではなくVref(4.5V、オペアンプ段と同じ仮想グラウンド)に変更**

これらはSPICEモデル特有の癖ではなく、実チップの差動入力段がV−レール際で正常動作しないという一般的なバイポーラ差動対の制約に起因すると考えられるため、実機でも同様の修正が必要と判断した。

## Phase 2: 手計算による検証と発見されたバグ（2026-07-01）

SPICEシミュレータ(ngspice/LTspice)がこの環境で即座に使える状態になかったため、データシートの式(gm≈19.2×IABC[A]、VBIAS=2VBE+V−等)を使った手計算での検証を実施。**この過程で実機が正しく動作しない可能性が高い設計バグを2つ発見し、修正した。**

### バグ1: OTA段の無音時ゲインが過大（RL=47kΩ, 旧R_ABC=33kΩ）
- 旧設計: IABC≈233µA → gm=19.2×233µA≈4.48mA/V → 電圧ゲイン=gm×RL=4.48m×47k≈**210倍(46dB)**
- ベース信号(passive想定で0.1〜0.5Vpk程度)を210倍すると21〜105Vppに相当し、9V単電源(Vref=4.5V中心)では物理的に出力し切れず、**圧縮が効く前にほぼ常時ハードクリップする**状態だった
- **対策**: R_ABCを33kΩ→**820kΩ**に変更。IABC≈9.3µA→gm≈179µS→無音時ゲイン≈**8.4倍(18.5dB)**に抑制。0.5Vpk入力でも4.2Vpk程度に収まり、9V電源内で現実的なヘッドルームに収まる

### バグ2: エンベロープ検出電圧がQ_ctrlのベースを駆動できない
- 旧設計: エンベロープノードは無信号時0V基準で、信号が来た時だけD1(1N4148, 順方向降下≈0.6V)経由でC_holdを充電する構成だった。ベース信号は控えめなレベルのことが多く、整流後の電圧(1Vpk入力でも≈0.4V程度)ではQ_ctrl(2SC1815, Vbe(on)≈0.6V)がほとんどオンにならず、**現実的な入力レベルでは圧縮がほぼかからない**可能性が高かった
- **対策**: R_prebias(3.3MΩ, V+→エンベロープノード)を追加し、無信号時のエンベロープノードを約0.56V（Q_ctrlのオン閾値のすぐ下）にプリバイアス。VR1(Sustain)が0の時は結合が切れゲイン変化なし、VR1を上げるほどこのプリバイアス電圧がQ_ctrlベースに伝わり、そこに信号由来の追加電圧が上乗せされることで、小さな信号変化にも敏感に反応するようになる

### 検証はしたが未解決・Phase3実機調整が必要な項目
- [ ] OTA無音時ゲイン8.4倍は概算目標値であり、実際のベース(パッシブ/アクティブ、指弾き/スラップ等)の出力レベルに応じてR_ABC・RLを微調整する必要がある可能性が高い
- [ ] Q_ctrlのVbe(on)や2SC1815の個体差により、R_prebiasの3.3MΩは±20%程度ずれる可能性がある。ブレッドボードでテスターを当てながら微調整すること

## Phase 2b: 実SPICEシミュレーション（LTspice）での検証（2026-07-01）

上記の手計算に続き、実際にLTspiceをインストールし(`winget install AnalogDevices.LTspice`)、LM13700の実トランジスタレベルSPICEモデル（[deanm1278/LM13700-spice-model](https://github.com/deanm1278/LM13700-spice-model)、TIの公開回路がベース）を使って`sim/`ディレクトリでトランジェント解析を行った。

### この検証で追加で見つかり、上記セクション3に反映した設計修正
1. **OTA差動入力へのアッテネータ追加**（1/63.5、R_atten_a/R_atten_b新設）— 手計算では見落としていた、データシート自身が明記する「OTA入力は数十mVppまでしか線形動作しない」という制約に、そのままでは1.6Vpp近い信号を突っ込んでいたことが判明
2. **OTA差動入力(pin3, pin4)の基準をGNDからVrefに変更** — GND直結だとSPICE上でゲインがほぼゼロになることを確認。V−レール際で差動入力トランジスタが実質遮断される
3. **R_ABCを75kΩに再修正**（IABC≈100µA） — SPICEでIABCを20µA〜5mAまで振ったところ、47kΩの負荷抵抗・9V電源の条件では500µA以上でOTA単体のゲインが電源電圧に張り付いて飽和することが分かり、飽和しない範囲に再設定。その分ゲインが控えめになるため、メイクアップゲイン段(5.7倍)と合わせてトータルで帳尻を合わせる設計とした

### 検証の限界（正直に記載）
- 上記1〜3の修正を反映した回路全体でのトランジェント解析（実際に「静かな信号→大きな信号→静かに戻る」を流して波形上で圧縮を確認する）は、**このSPICEモデル特有の収束の癖により最終確認できていない**。具体的には、OTA入力をコンデンサでACカップリングした状態（＝実機と同じ、必須の構成）にすると、SPICEのソルバー（Newton法・Gminステッピング）が物理的に妥当でないと思われる別解に収束してしまう現象が再現性をもって発生し、初期条件の指定(`.ic`/`uic`)やバイアス抵抗値の調整でも解消できなかった
- 一方、OTA単体を理想電圧源で直接駆動した簡易テスト（`sim/isolate_test9.cir`等）では上記の修正方針（Vrefバイアス、適切なIABC範囲）で正常にゲインが出ることを確認済み。**上記1〜3の設計修正自体はデータシートの記述および単体テストの両方と整合しており、妥当性は高いと考えている**
- 結論: **今回のSPICE検証は「新たな2つの設計不備を発見・修正する」という実質的な成果を上げたが、修正後の全体波形での動作確認は未完了。** 最終的な動作確認はPhase3のブレッドボード実測に委ねる
- 関連ファイル: `sim/bass_compressor_test2.cir`（本回路のSPICEネットリスト）、`sim/isolate_test*.cir`（切り分け用の単体テスト群）、`sim/lm13700.sub`（使用したLM13700モデル）

### RL1（DPDT）の正しい配線（修正版）

前バージョンでは「1系統のみ使用、もう1系統は予備」としていたが、これだと未使用側（コンプレッサー回路の出力）が常時出力ジャックに繋がったままになり、バイパス時もコンプレッサー側の出力段が信号経路に乗り続けてしまう（＝本当のトゥルーバイパスになっていない）。DPDTの2系統を両方使い、入力側・出力側を同時に切り替える構成に修正する。

- **Pole 1（入力切替）**: COM=入力信号(J1後段) / NO=C_in（コンプレッサー回路へ） / NC=ノードDRY（バイパス時の直通経路）
- **Pole 2（出力切替）**: COM=出力信号(J2へ) / NO=C5後段（コンプレッサー回路の出力） / NC=ノードDRY（Pole1のNCと同じノード）
- リレー非励磁（バイパス時）: 入力→NC→ノードDRY→NC→出力 の直通経路のみが繋がり、コンプレッサー回路は入力・出力とも完全に切り離される（正しいトゥルーバイパス）
- リレー励磁（エフェクトON時）: 入力→NO→コンプレッサー回路→C5→NO→出力 の経路のみが繋がる
