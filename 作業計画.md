# ベース用コンプレッサー 開発計画

作成日: 2026-07-01
目的: ベースギター用コンプレッサーペダルの回路設計 → 試作検証 → KiCad基板設計 → JLCPCBへの基板製造依頼

## 前提（今回決定した方針）

- 回路方式: **OTA方式**（Ross Compressor / Dyna Compタイプ。LM13700ベース）
  - 理由: 部品点数が少なく安価、実績豊富、部品調達がしやすい（CA3080は生産終息傾向のため、入手性の良い**LM13700**（デュアルOTA）を採用）
- JLCPCB依頼範囲: **基板製造のみ（Fab）**。実装（PCBA/SMT組立）は依頼せず、部品は自分でLCSC/秋月/Mouser等から調達し手ハンダする
  - → 部品はスルーホール（THT）中心で設計し、手ハンダしやすくする

## 確定仕様（2026-07-01 決定）

- **電源**: 9V電池駆動のみ（DCジャックなし。電池スナップ直結、必要ならバッテリークリップ用の基板パターンを用意）
- **LED**: あり（バイパス状態インジケーター、後述のラッチ回路の状態を表示）
- **対象音域**: 4弦ベース前提（低音E ≈ 41Hz）。入力ハイパスのカットオフはこれを減衰させない値（目安 10Hz台〜20Hz程度）に設定し、エンベロープ検出側の時定数も4弦の基音を追従できるレンジで調整する
- **筐体**: 1590B。ただし通常の「パネルにポット/スイッチを取り付けて配線」ではなく、**基板を露出させるデザイン**（1590Bに基板がそのまま収まる/窓から見える構成、パネル配線レス）
  - ⚠️**2026-07-01更新**: 基板が100.1×132.1mmに拡大し1590Bには収まらないことが判明。**ユーザー判断によりケース自体を今回の調達対象から除外**（基板剥き出し運用、ケース選定は別途後日。詳細はPhase7参照）
- **ノブ構成**: 最小限の **Sustain（圧縮量）+ Level（出力）の2ノブ**。Attack/Releaseは回路内で固定値（Phase 2/3で追い込む）
- **ポット**: 基板実装型ロータリーポット（Alpha RD901F等、指でつまみを回せるPCBマウントタイプ）を採用し、配線レスで基板に直付け。露出デザインと相性の良い部品を選定
- **オン/オフスイッチ（バイパス）**: 通常のフットスイッチではなく **Cherry MX準拠のキースイッチ**を使用
  - Cherry MXはモーメンタリ（押している間だけON）のため、そのままではトゥルーバイパスのラッチ動作ができない
  - → **タクトスイッチ的な入力として使い、CMOSフリップフロップ（4013等）でラッチしてリレーを駆動する「ソフトタッチ・トゥルーバイパス」回路**を追加する（信号経路自体はリレーが機械的に直接切り替えるためトゥルーバイパス）
  - LEDはこのフリップフロップの状態を表示するドライブ回路と共用
  - Cherry MX用のPCBフットプリント（5pin/plate mount想定）をKiCadで用意する

## 作業フェーズ

### Phase 0: 仕様確定 ✅ 完了（2026-07-01）
- ノブ構成・ポット種別・バイパススイッチ方式・電源・LED・筐体・対象音域を確定（上記「確定仕様」参照）
- 参考回路（Ross Compressor の解析記事: Electrosmash、General Guitar Gadgetsの配布回路図等）を突き合わせてベース向けに調整する定数（入力カップリングコンデンサ容量、エンベロープの時定数）の方針を決める

### Phase 1: 回路設計（部品レベルの仕様確定） ✅ 完了（2026-07-01、`docs/circuit_reference.md`）
LM13700・TL072・TC4013のピン配置をデータシート/KiCad標準シンボルで検証したうえで、9V電池単電源・Sustain+Levelの2ノブ・Cherry MX+フリップフロップ+リレーによるソフトタッチ・トゥルーバイパスまで含めた部品レベルの回路を確定。レビュー中にバイパス回路の設計不備（リレー1系統のみ使用でバイパス時に完全分離できていなかった）を発見し、DPDT両系統を使う正しいトゥルーバイパス構成に修正済み。
- **コンプレッサー本体**: OTA(LM13700) + デュアルオペアンプ(TL072等、ノイズ・入手性で選定) によるゲインリダクション段、整流（エンベロープ検出）段、入出力バッファの回路構成を確定。ノブはSustain/Levelの2つのみに絞った構成にする
- **バイパス部**: Cherry MXキースイッチ（モーメンタリ入力）→ デバウンス → CMOSフリップフロップ（4013等）→ 小信号リレー駆動 によるソフトタッチ・トゥルーバイパス回路、およびLEDドライブを設計
- 電源は9V電池単電源、仮想グラウンド（半電源バイアス）方式。電池残量を考慮した低消費電流設計（リレー・LED電流に注意）
- 部品はJLCPCB提携部品庫（LCSC）での在庫・入手性を確認しながら選定。ポットは基板実装型ロータリーポット（Alpha RD901F等）、スイッチはCherry MX用フットプリントを使用

### Phase 2: シミュレーション検証（LTspice / ngspice）
- LM13700のSPICEモデルを用いてゲインリダクション動作、アタック/リリース特性をシミュレーションで確認
- 実機を組む前にここで論理的な誤りを潰す（コンプレッサーは帰還の極性を間違えるとエキスパンダー的挙動になるため要注意）

### Phase 3: ブレッドボード/ユニバーサル基板での試作
- 実際にベースを繋いで圧縮感・ノイズ・発振の有無を確認
- 定数（アタック/リリースの時定数、コーナー周波数）を実測しながらベース向けに追い込む
- **この段階で「ちゃんと圧縮する」ことを確認してから基板発注に進む**（基板を先に発注して動かないと手戻りが大きいため）

### Phase 4a: KiCad回路図の作図・ERC ✅ 完了（2026-07-01）
`kicad/bass_compressor/` にKiCad 8プロジェクトを作成し、Phase1の設計を回路図に反映。ERC（電気ルールチェック）はエラー0件（軽微なシンボル警告3件のみ、電気的には無害と確認済み）。KiCad BOM出力(`bom_kicad.csv`)も`bom/部品リスト.md`の内容と数量・値が一致することを確認済み。

### Phase 2: 回路検証（手計算 + 実SPICE） ✅ 実施（2026-07-01、`docs/circuit_reference.md`内「Phase 2」「Phase 2b」節）
- **手計算**: データシートの式で検証し、無音時ゲイン過大（R_ABC 33k→820k）、エンベロープ電圧不足（プリバイアス抵抗3.3MΩ追加）の2バグを発見・修正
- **実SPICE(LTspice)**: `winget install AnalogDevices.LTspice`でインストールし、LM13700の実トランジスタレベルSPICEモデルで検証（`sim/`ディレクトリ）。さらに2バグを発見・修正: (1)OTA差動入力へのアッテネータ不足（データシート上限の数十mVppを大幅超過していた）、(2)OTA差動入力の基準電位がGNDだと事実上遮断されゲインがほぼ出ない。ついでにR_ABCも75kΩに再調整（500µA以上でゲイン飽和と判明）
- 上記すべて回路図・BOMに反映済み（**47部品、ERC 0エラー**）
- **正直な限界**: 修正後の回路全体の波形シミュレーション（圧縮動作の最終確認）は、このSPICEモデル特有の収束の癖（ACカップリング構成で解が不安定になる）により完了できていない。個別ブロックの単体テストでは修正方針の妥当性は確認済み。**最終的な動作確認はPhase3のブレッドボード実測が必須。**

### Phase 4b: KiCad基板レイアウト設計 ✅ 完了（2026-07-01）
**注意**: ユーザー判断によりPhase3(ブレッドボード実測)を飛ばして先行着手。回路定数（R_ABC等）は未実測のため、試作後に値の修正・レイアウトの手戻りが発生する可能性がある点は了承済み。

- `kicad/bass_compressor/bass_compressor.kicad_pcb` 完成。独自Pythonスクリプト（グリッド配置＋A*迷路ルーター）で47部品を自動配置・自動配線
- **DRC: エラー0件、未配線0件**（警告51件はすべて`lib_footprint_mismatch`＝手書きフットプリント定義が正式ライブラリと完全一致しないという情報的な警告で、電気的問題ではないため許容）
- 基板イメージ `bass_compressor_board.svg` 出力済み
- **⚠️要確認事項**: 基板サイズが当初想定の100×50mmから**100×190mmまで拡大**している（47点のTHT部品・ICソケット3個・リレー・6.35mmジャック2個を衝突なく収めるため）。ユーザー了承のもと、いったんこのサイズでDRC/配線を完了させた。**TD6-11-3N（外形115.5×64.5×31.8mm）には高さが大幅に超えるため収まらず、後日ケースサイズの見直しか部品密度の見直し（IC直付け化・SMD化・基板分割等）が必要**
- **原因判明（2026-07-01）**: 抵抗の配置座標を確認したところ、全部品が**13mm×14mmの均一グリッド**に配置されていた。抵抗自体の実サイズ（パッド間隔10.16mm）よりかなり広く、ICソケット/リレー/ジャックなど最大サイズの部品に合わせた間隔を全部品に一律適用しているのが基板肥大化の直接原因。部品サイズに応じて詰める配置に変えれば大幅に縮小できる見込み
- **今後の進め方（ユーザー方針）**: まず現在のレイアウトで「部品を乗せた基板」の3Dレンダリング画像を生成してみて、そこで気づいた点を反映してレイアウトを変更し、**同じ手順（配置→配線→DRC→レンダリング）でもう一度やり直す**というイテレーションを予定
- **レイアウト詰め直し 実施済み（2026-07-01）**: 各部品の実パッド範囲（シルク上のbounding boxではなく実際の銅箔占有域）を測定し直し、行間・列間の間隔を部品サイズに応じて詰めた上で、独自A*グリッドルーター（1mm格子、2層+ビア、パッド1個ずつの実サイズに応じた円形クリアランスハローを持たせた版）で全結線を再配線
  - **基板サイズ: 100×190mm → 100×121mm（36%縮小）**。IC1(LM13700, DIP-16ソケット)のピン列がボディ中心から大きく片側にオフセットしている（対称なbounding boxの想定が誤りだった）ことが判明し、リレー(RL1)との実クリアランスを再計算して衝突を回避
  - **DRC: エラー0件、未配線0件**（警告51件は従来通りlib_footprint_mismatchのみで電気的に無害）
  - `bass_compressor_board.svg` / `bass_compressor_preview.pdf` / `drc_report.json` を更新済み
  - まだ100×190mm→100×121mmへの縮小に留まり、TD6-11-3N(115.5×64.5mm)には引き続き収まらない。さらに詰めるには部品密度そのものを下げる方策（IC直付け化・SMD化・基板分割等、Phase4bで既出の論点）が必要
  - **3Dレンダリング解決（2026-07-01）**: `kicad-cli pcb export glb`はデフォルトだとVRML(.wrl)モデルが読み込めず部品が表示されない環境依存の問題があったが、`--subst-models`（同名のSTEPモデルに差し替え）オプションと`KICAD8_3DMODEL_DIR`環境変数の明示指定で解決。またKiCad標準ライブラリに3DモデルがないCherry MXキースイッチ+キーキャップ、ロータリーポット(VR1/VR2)は、Blenderレンダースクリプト側で簡易プレースホルダー（箱・円柱）を実座標に合わせて追加する方式で表示
  - **操作パネルレイアウト変更（2026-07-01, ユーザー指示）**: VR1(Sustain)・VR2(Level)を上部に並べ、その間にLED1を配置。SW1(バイパススイッチ)は基板最下部の新しい行に移動（GND1本+IC3への2点間ネットNODE_B1本のみで複雑な分岐がなく、長距離配線でも問題にならなかった）。再配置→再配線→DRCを実行し**エラー0件・未配線0件を確認**。SW1用の新しい行を最下部に追加したため基板サイズは100×121mm→**100×132mm**に微増
  - **KiCad上での手動微調整＋再配線（2026-07-01）**: ユーザーがKiCad GUI上でLED1/SW1/VR1/VR2の位置を微調整。既存配線をいったん全削除して同じA*ルーターで再配線したところGNDの1点(R2ピン2)だけ経路が見つからなかったが、ネット処理順序でGND(最大・32パッド)を最優先に回すよう変更したところ解消。**DRC: エラー0件・未配線0件**（配置は変更していないため基板サイズ100×132mmは変わらず）
- 試作で確定した回路をKiCad 8で回路図化（インストール確認済み: KiCad 8.0.1, kicad-cli あり）
- ERC（電気ルールチェック）を実行しエラーゼロを確認
- **基板露出デザイン**のため、1590Bの内寸に収まる外形サイズで設計し、ポット（Alpha RD901F）とCherry MXキースイッチの軸/キーキャップが1590Bの窓・開口部から出る位置にレイアウト
- 基板外形はねじ止め用の取付穴（1590Bの固定方法に合わせる）も考慮
- PCBレイアウト、DRC（デザインルールチェック）実行

### Phase 5: DFM（製造性）チェック ✅ 完了（2026-07-01）
JLCPCB公式capabilitiesページ（2-layer, 標準FR-4）の数値と、実際の基板データをスクリプトで測定した値を突き合わせて確認。**全項目、標準スペック内で余裕あり。設計変更なし**。

| ビア（パッド径/ドリル径） | 最小0.25mm / 0.15mm | 0.6mm / 0.3mm | ✅ 余裕あり（極小穴の追加費用ゾーンにも入らない） |

### Phase 6: ガーバー出力・JLCPCB発注（ガーバー生成まで完了、発注はユーザー実施）
- `kicad-cli`でガーバー(F/B Cu, F/B Silkscreen, F/B Mask, Edge.Cuts)・ドリル(PTH/NPTH分離)・ジョブファイルを生成し、`kicad/bass_compressor/gerbers/`と`bass_compressor_gerbers.zip`に出力済み
- KiCadから「JLCPCB用ガーバー出力」プラグイン or 手動プロットでガーバー/ドリルファイルを生成しZIP化
- JLCPCBサイトにアップロードし、以下を指定して発注
  - 枚数（試作は5枚が最小ロット）
  - 層数2層、板厚1.6mm、レジスト色、シルク色、表面処理（HASL推奨・安価）
  - 実装は依頼しない（PCBAオプションはOFF）のままチェックアウト
- 送料・納期を確認して発注確定（実際の支払い・発注操作はユーザー本人が実施）

### Phase 7: 部品調達（基板以外） ✅ 秋月での購入可否チェック完了（2026-07-01, 詳細は `bom/部品リスト.md`）
- **秋月電子で購入**: TL072相当（`TL072G-D08`/`NJM072D`）、4013相当（`TC4013BP`）、1N4148、単連10kΩ基板用ロータリーポット（`RK09D117000B/C`, ※Alpha RD901Fの直接代替品）、5VリレーDPDT（`941H-2C-5D`）、スイッチ付き6.3mmジャック（`MJ-188LPD-8C`, 電池ON/OFF兼用）、抵抗/コンデンサ/LED/ICソケット/電池スナップ等一般部品
- **秋月では買えない分はAmazon.co.jpでまとめて調達（Mouser発注は基本不要）**: LM13700N、Alpha RD901F（必要な抵抗値そのままで購入可）。ただしLM13700はマーケットプレイス品の品質リスクがあるため、信頼性優先なら1点だけMouser日本の正規品`LM13700N/NOPB`にする選択肢も残す
- **キースイッチ（Cherry MX互換）**: ユーザー手持ち品を使用のため調達不要
- JFET（2N5457相当）はPhase0で保険的に検討していたが、Phase1で確定した最終回路には未使用と判明済み（調達不要）
- **⚠️筐体方針変更（2026-07-01）**: Phase4bのレイアウト詰め直しで最終基板サイズが**100.1×132.1mm**に確定し、当初想定の1590B代替`TD6-11-3N`（115.5×64.5mm）には収まらないことが判明。代替候補としてタカチ`TD12-18-5N`（外形120×180×50mm）を検討したが、**ユーザー判断によりケース自体を今回の調達対象から除外**（基板剥き出し運用、ケース選定は別途後日）。部品調達ルートは「秋月＋Amazon.co.jp」の2箇所構成のまま変更なし（詳細は`bom/部品リスト.md`）
