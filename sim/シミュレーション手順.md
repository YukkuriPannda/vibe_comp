# 波形シミュレーション手順

コンプレッサー回路全体の波形（エンベロープ追従・ゲインリダクション・アタック/リリース）を確認するための、再現可能な手順。

## 環境

- ツール: **LTspice 26.0.2 for Windows**（インストール済み: `C:\Users\Pannda\AppData\Local\Programs\ADI\LTspice\LTspice.exe`）
- 過去の`.log`を見る限り、このLTspice環境は既にASCII rawファイル出力（`AsciiRawFile = true`）が有効になっている。ASCII raw はバイナリrawよりパースが簡単なので、このままでよい。

## 対象ファイル

- `sim/bass_compressor_test2.cir` — 現状もっとも回路の実態（`bom_kicad.csv`のR1-R21, C1-C11の定数）に近い、完成形の全体回路netlist。**回帰チェックはこれを正とする。**
- `sim/lm13700.sub` — OTA（LM13700系）のSPICEサブサーキットモデル。IC1をNJM13700Dに変更したが電気的に完全互換のため、モデルはこのままで良い。
- `sim/isolate_test*.cir`, `sim/sweep_gm.cir` などはOTAの動作を切り分けるためのデバッグ用中間ファイル。正式な検証には使わない（参考記録として残っているだけ）。

## 実行手順

1. 回路定数を変更した場合は、まず `bass_compressor_test2.cir` を実際の回路（R/C値、`.param SUSTAIN_FRAC`/`LEVEL_FRAC`など）に合わせて更新する。
2. LTspiceをバッチモードで実行し、`.raw`/`.log`を生成する。
   ```
   & "C:\Users\Pannda\AppData\Local\Programs\ADI\LTspice\LTspice.exe" -b "D:\OtherProjects\vibe_comp\sim\bass_compressor_test2.cir"
   ```
3. `.log`を確認する。「Direct Newton iteration failed to find operating point」が出ても「Gmin stepping succeeded in finding the operating point」が続けば実用上問題ない（これは今回のOTA回路で毎回出る既知の挙動）。逆に収束自体に失敗している場合は回路定数か初期条件(`.ic`)を見直す。
4. `.raw`を解析・プロットする。現状、専用のパース/プロットスクリプトがリポジトリに存在しないため、新規に `sim/plot_raw.py` を作成することを推奨する（Python + `numpy` でASCII rawを読み、`matplotlib`で主要ノードをプロットする程度で十分）。LTspice GUIで直接`.raw`を開いて目視確認するだけでも良い。

## 確認すべき波形・指標

`bass_compressor_test2.cir` の入力は「静か(0.05Vpk) → 50msで急に大音量(0.8Vpk) → 250msで静かに戻る」というPWLエンベロープ（`Venv`）に100Hzサイン波を乗せたもの（`Bsig`）。

- **ゲインリダクション**: `OUT1`（OTA出力）と`OUTFINAL`（最終出力）で、入力振幅比(0.05→0.8、16倍)に対して出力振幅比がどれだけ圧縮されているか。
- **アタックタイム**: 50ms時点の急変後、ゲインが下がりきるまでの時間。回路定数からの概算値は `Rattack(2.2k) × Chold(1u) ≈ 2.2ms`。
- **リリースタイム**: 250ms時点で静かに戻った後、ゲインが戻るまでの時間。概算値は `Rrelease(220k) × Chold(1u) ≈ 220ms`。
- **SUSTAIN_FRAC・LEVEL_FRACのスイープ**: `.param`を`.step param`に変えて0〜1の範囲を振り、Sustain/Levelつまみの効き方が単調で破綻していないか確認する。
- **歪み・クリップ**: `OUTFINAL`が電源レール付近でクリップしていないか。

## 今回のBOM・footprint変更に伴う注意点

- **IC1 (LM13700→NJM13700D)**: 電気的に完全互換のため、SPICEモデル変更・再シミュレーションは不要。
- **RL1・J1のfootprint修正、コンデンサのリードピッチ変更(P2.00mm→P5.00mm)**: いずれも実装上の変更であり、回路定数(R/C値)や電気的トポロジーには影響しないため、これ単体では再シミュレーション不要。
- **ただし**、footprint修正エージェントによる再配線作業でネットの繋ぎ間違い（誤配線）が起きるリスクはゼロではない。再配線が完了したら、**ERC（回路図側）に加えて、本シミュレーションも一度再実行して波形が壊れていないことを確認するのが安全**。
